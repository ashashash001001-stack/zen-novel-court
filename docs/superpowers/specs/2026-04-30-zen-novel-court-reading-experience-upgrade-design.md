# Zen Novel Court 閱讀體驗升級 Roadmap 設計

> **項目**：Zen Novel Court（小說閣）
> **類型**：Astro 靜態網站應用
> **版本**：1.0
> **日期**：2026-04-30

---

## 1. 項目概述

### 1.1 目標

全方位提升網站的閱讀體驗，同時強化搜尋、收藏、部署、SEO、収益化等功能，打造一個讓用戶沉浸其中願意長期使用的小說閱讀平台。

### 1.2 核心理念

「先讓看書這件事變成一種享受，再談商業化。」

### 1.3 現有基礎

- Astro 6.1.5 + TypeScript 靜態網站
- 首頁、書庫、書籍詳情、章節閱讀頁面
- localStorage 書籤與進度追蹤
- 簡繁體 i18n 自動轉換
- 響應式設計 + PWA + SEO

## 3. 第一階段：Month 1 - 閱讀介面大改版

### 3.1 字體系統

**功能描述**：
- 可選擇多種中文字體（系統字體、Web Font）
- 調整字體大小（滑桿：12px - 24px）
- 調整字體粗細（細體、普通、粗體）

**實作要點**：
- 使用 CSS custom properties 管理字體變數
- 支援 localStorage 儲存用戶偏好
- 提供合理的預設值（16px，正常粗細）

**驗收標準**：
- [ ] 用戶可調整字體大小
- [ ] 用戶可選擇字體
- [ ] 設定會保存並在下次訪問時套用

---

### 3.2 排版優化

**功能描述**：
- 調整行高（1.5 - 2.0）
- 調整段落間距
- 支援兩端對齊（justify）

**實作要點**：
- 在 ReaderLayout 中新增排版控制面板
- 使用 CSS variables 控制所有排版參數
- 行動端預設使用不兩端對齊（避免文字過密）

**驗收標準**：
- [ ] 行高可調整
- [ ] 段落間距可調整
- [ ] 桌面端可選擇是否兩端對齊

---

### 3.3 翻頁體驗

**功能描述**：
- 優化點擊區域（左中右區域控制前後翻頁）
- 支援側邊滑動翻頁（觸控裝置）
- 虛擬滾動支援（長篇章節）

**實作要點**：
- 左側 30% → 上一章
- 中間 40% → 顯示/隱藏控制面板
- 右側 30% → 下一章
- 觸控滑動支援使用 Hammer.js 或原生 touch 事件

**驗收標準**：
- [ ] 點擊正確區域可翻頁
- [ ] 滑動翻頁在觸控裝置正常運作
- [ ] 滑動方向正確（向左滑下一章，向右滑上一章）

---

### 3.4 TOC 目錄

**功能描述**：
- 浮動目錄側邊欄（可收起/展開）
- 章節列表可點擊跳轉
- 書籤功能（標記重要章節）

**實作要點**：
- 使用 slide-in 動畫展示目錄
- 書籤資料存儲於 localStorage
- 當前章節高亮顯示

**驗收標準**：
- [ ] 目錄可展開/收起
- [ ] 點擊章節可跳轉
- [ ] 書籤功能正常運作
- [ ] 當前章節有視覺標示

---

### 3.5 背景主題

**功能描述**：
- 預設主題：紙張白（暖白）
- 護眼模式：淡綠/淡棕
- 可自訂背景色

**實作要點**：
- 主題使用 CSS custom properties
- 支援主題預覽（即時套用不需儲存）
- 主題選擇存儲於 localStorage

**驗收標準**：
- [ ] 至少 3 種主題可選擇
- [ ] 主題切換無需重新載入頁面
- [ ] 主題偏好會被記憶

---

## 4. 第二階段：Month 2 - 互動與統計

### 4.1 黑暗模式

**功能描述**：
- 系統自動切換（根據 OS 設定）
- 手動切換（淺色/深色/自動）
- 深色模式專屬配色（不是單純反相）

**實作要點**：
- 使用 CSS media query (`prefers-color-scheme`)
- 手動切換覆蓋自動設定，存於 localStorage
- 深色模式需要有專屬的閱讀背景（深灰而非純黑）

**驗收標準**：
- [ ] 支援系統自動切換
- [ ] 支援手動切換
- [ ] 黑暗模式閱讀體驗舒適

---

### 4.2 閱讀統計

**功能描述**：
- 本章字數顯示
- 預估閱讀時間（根據平均閱讀速度）
- 總進度百分比（本章在書籍中的位置）

**實作要點**：
- 字數統計在 build time 計算
- 閱讀速度預設 300 字/分鐘，可調整
- 顯示剩餘章節數

**驗收標準**：
- [ ] 顯示本章字數
- [ ] 顯示預估閱讀時間
- [ ] 顯示進度百分比

---

### 4.3 進度追蹤

**功能描述**：
- 自動記憶當前閱讀章節
- 顯示進度條（可點擊跳轉）
- 閱讀歷史記錄

**實作要點**：
- 使用 localStorage 儲存 `{ bookId: chapterId }`
- 進度條使用 `<progress>` 或自訂 div
- 歷史記錄限制保留最近 50 筆

**驗收標準**：
- [ ] 重新打開書籍會自動跳轉到上次位置
- [ ] 進度條正確顯示閱讀進度
- [ ] 點擊進度條可跳轉

---

### 4.4 章節預載

**功能描述**：
- 智慧預載下一章內容
- 離線緩存已讀章節

**實作要點**：
- 使用 Astro 的 experimental cache 或自訂實作
- 只預載下一章，不預載全書
- 考慮用戶網路狀況（閒置時預載）

**驗收標準**：
- [ ] 下一章載入時間 < 100ms（有預載時）
- [ ] 不會預載過多章節浪費流量

---

## 5. 第三階段：Month 3 - 功能強化

### 5.1 進階搜尋

**功能描述**：
- 標籤搜尋（多標籤篩選）
- 作者搜尋
- 熱門關鍵詞顯示
- 搜尋歷史

**實作要點**：
- 使用 Fuse.js 做客戶端搜尋（數據量小）
- 搜尋結果即時過濾
- 熱門關鍵詞可 hardcode 或從分析數據產生

**驗收標準**：
- [ ] 標籤搜尋功能正常
- [ ] 作者搜尋功能正常
- [ ] 搜尋結果正確顯示
- [ ] 無結果時顯示友好提示

---

### 5.2 收藏強化

**功能描述**：
- 收藏夾分類（自訂資料夾）
- 筆記功能（針對章節寫筆記）
- 標籤化收藏

**實作要點**：
- localStorage 結構：`{ favorites: [{ bookId, folder, tags, notes }] }`
- 支援拖曳排序
- 筆記支援 Markdown 語法

**驗收標準**：
- [ ] 可建立收藏夾
- [ ] 可對章節寫筆記
- [ ] 收藏可加上標籤
- [ ] 收藏夾可排序

---

### 5.3 書架優化

**功能描述**：
- 多種排序方式（最近閱讀、收藏時間、標題）
- 篩選功能（分類、標籤、狀態）
- 封面顯示（如果有的話）

**實作要點**：
- 書架頁面需要 client-side 互動（Astro Island）
- 使用 Preact 或 React 做互動元件

**驗收標準**：
- [ ] 支援多種排序
- [ ] 支援篩選
- [ ] 書架載入速度 < 1s

## 8. 技術架構

### 8.1 前端技術

| 技術 | 用途 |
|------|------|
| Astro | 靜態網站框架 |
| TypeScript | 類型安全 |
| CSS Custom Properties | 主題系統 |
| Preact | 互動元件（書架、搜尋） |
| localStorage | 用戶資料持久化 |
| Fuse.js | 客戶端搜尋 |

### 8.2 數據結構

```typescript
interface UserPreferences {
  fontSize: number;        // 12-24
  fontFamily: string;
  lineHeight: number;      // 1.5-2.0
  theme: 'light' | 'dark' | 'sepia' | 'green';
  autoTheme: boolean;      // 跟隨系統
}

interface ReadingProgress {
  bookId: string;
  chapterId: number;
  timestamp: number;
  progress: number;        // 0-100
}

interface Favorite {
  bookId: string;
  folder?: string;
  tags: string[];
  note?: string;
  addedAt: number;
}
```

## 9. 風險與緩解

| 風險 | 緩解措施 |
|------|----------|
| 功能過多導致開發停滯 | 嚴格按照階段優先級執行 |
| localStorage 空間不足 | 定期清理歷史記錄，限制資料量 |
| 黑暗模式閱讀不舒適 | 參考成熟閱讀 app 的配色 |
| SEO 改動影響現有排名 | 逐步改動，監控流量變化 |

## 11. 總結

本設計涵蓋了 Zen Novel Court 未來 6 個月的開發方向，以閱讀體驗為核心，逐步擴展到功能強化、基礎設施優化，最終實現商業化可能。

3. **數據驅動** — 持續追蹤優化效果
4. **商業可行** — 為未來収益做準備
